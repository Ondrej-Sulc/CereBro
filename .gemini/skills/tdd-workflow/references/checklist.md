# TDD Cycle Checklist

## Phase 1: Planning
- [ ] Behavior or bug clearly identified.
- [ ] Public API/Interface defined.
- [ ] Test cases prioritized.
- [ ] User approval for interface changes (if any).

## Phase 2: RED
- [ ] Exactly ONE new test case written.
- [ ] Test fails for the right reason (e.g., expected "X" got "undefined").

## Phase 3: GREEN
- [ ] Minimal code implemented.
- [ ] Test passes.
- [ ] No extra features or speculative code added.

## Phase 4: REFACTOR
- [ ] Code is green (all tests pass).
- [ ] Redundant code removed.
- [ ] Variable/function names improved.
- [ ] Logic simplified.
- [ ] All tests still pass after refactoring.

## Phase 5: Repeat
- [ ] Ready for the next test case.
- [ ] Overall system integrity maintained.
